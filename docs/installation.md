# Installation

`vault` ships pre-built binaries for macOS (x86_64 + aarch64) and Linux (x86_64 + aarch64, musl). Windows is not currently supported.

## Shell installer (recommended)

The hosted installer downloads the right binary for your platform, places it on your `PATH`, and installs shell completions and the man page where they need to go:

```bash
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/dbtlr/vault-cli/releases/latest/download/vault-cli-installer.sh \
  | sh
```

If you prefer to inspect the installer before running it (recommended for any `curl | sh`-style install):

```bash
curl -LsSf \
  https://github.com/dbtlr/vault-cli/releases/latest/download/vault-cli-installer.sh \
  -o vault-cli-installer.sh
less vault-cli-installer.sh
sh vault-cli-installer.sh
```

The installer is generated by [cargo-dist](https://github.com/axodotdev/cargo-dist). It modifies your shell profile (`~/.zshrc`, `~/.bashrc`, or `~/.config/fish/config.fish`) to add the install directory to `PATH`. Re-source your shell or open a new terminal afterward.

## Cargo

If you have a Rust toolchain installed, build from source:

```bash
cargo install --git https://github.com/dbtlr/vault-cli vault-cli
```

Or pin a specific tag:

```bash
cargo install --git https://github.com/dbtlr/vault-cli --tag v0.26.0 vault-cli
```

`vault` requires the minimum supported Rust version (MSRV) declared in `crates/vault-cli/Cargo.toml` (`rust-version`). The project tracks latest stable.

## Manual binary download

Releases include pre-built archives at:

`https://github.com/dbtlr/vault-cli/releases/`

Each release ships `vault-cli-<version>-<target>.tar.gz` containing the `vault` binary, shell completions (`vault.bash`, `_vault`, `vault.fish`), and the man page (`vault.1`). Extract, place `vault` on your `PATH`, and copy the completions and man page into your system's appropriate directories.

## Verification

After install, verify the binary runs and that fixture validation produces output:

```bash
vault --version
vault --help
git clone https://github.com/dbtlr/vault-cli
cd vault-cli
vault -C fixtures/basic validate --summary --format json | head
```

You should see a finding summary as JSON with a non-zero `total`. If you see a panic, a missing-binary error, or no output, file an issue with the platform and shell you're using.

## Troubleshooting

- **`vault: command not found` after install.** The installer added the binary directory to your shell profile. Open a new terminal, or run `source ~/.zshrc` (or `~/.bashrc`).
- **Completions not loading.** Restart your shell. zsh users may need `compinit` if it's not already in their config.
- **macOS quarantine warning.** Binaries downloaded outside the Mac App Store may be quarantined. Run `xattr -d com.apple.quarantine /path/to/vault` once.
- **Cargo build fails with MSRV error.** Update your Rust toolchain to the latest stable: `rustup update stable`.

For developer-facing builds (cloning, contributing), see [development.md](development.md).
