# Installation

`vault` ships pre-built binaries for macOS (x86_64 + aarch64) and Linux (x86_64 + aarch64, musl). Windows is not currently supported.

## Shell installer (recommended)

The fastest path is to pipe the release installer into `sh`:

```bash
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/dbtlr/vault-cli/releases/latest/download/vault-cli-installer.sh \
  | sh
```

The installer downloads the right binary for your platform, places it in
`~/.cargo/bin` (or `$CARGO_HOME/bin` if set), and installs the shell
completions and man page next to it.

### Safer: download first, inspect, then run

If you would rather review the installer before executing it (always a
good idea for `curl | sh`-style installers), download it and run it as a
second step:

```bash
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/dbtlr/vault-cli/releases/latest/download/vault-cli-installer.sh \
  -o vault-cli-installer.sh
less vault-cli-installer.sh   # review
sh vault-cli-installer.sh
```

The installer is generated by [cargo-dist](https://opensource.axo.dev/cargo-dist/);
the source code that produces it is open and reproducible from the tag.

## Cargo

If you have a Rust toolchain installed, build from source:

```bash
cargo install --git https://github.com/dbtlr/vault-cli vault-cli
```

Or pin a specific tag:

```bash
cargo install --git https://github.com/dbtlr/vault-cli --tag v0.26.1 vault-cli
```

`vault` tracks the latest stable Rust toolchain (see `mise.toml`). See `docs/development.md` for the full MSRV policy.

## Manual binary download

Releases include pre-built archives at:

`https://github.com/dbtlr/vault-cli/releases/`

Each release ships `vault-cli-<target>.tar.xz` containing the `vault` binary, shell completions (`vault.bash`, `_vault`, `vault.fish`), and the man page (`vault.1`). Extract, place `vault` on your `PATH`, and copy the completions and man page into your system's appropriate directories.

## Shell completions

After install, enable completions for your shell with:

```bash
vault completions install
```

`vault completions install` auto-detects your shell from `$SHELL` and writes the right configuration. Supported shells: **bash**, **zsh**, **fish**, **powershell**, **elvish**, **nushell**.

To preview what it would write without modifying anything:

```bash
vault completions install --print
```

To replace an existing install (after upgrading vault, or to reset a manual edit):

```bash
vault completions install --force
```

To install for a specific shell rather than the one detected:

```bash
vault completions install zsh
```

### After a vault upgrade

The bash, zsh, powershell, and elvish installers write an `eval` line that re-evaluates the current `vault completions init <shell>` output every shell startup. New completions pick up automatically.

The fish and nushell installers write a script file (because those shells load completions from files, not eval). After upgrading vault, re-run `vault completions install fish` or `vault completions install nushell` to refresh the script with the new vault's completions.

### macOS bash target file

On macOS, the bash installer prefers `~/.bash_profile` over `~/.bashrc` when the former exists, since macOS launches Terminal sessions as login shells. If you use a non-standard bash setup, override the target with `BASH_ENV=/path/to/file vault completions install bash`.

### Manual install

If you prefer to wire completions in by hand, `vault completions init <shell>` emits the script to stdout. Pipe it where your shell expects it:

```bash
# bash
echo 'eval "$(vault completions init bash)"' >> ~/.bashrc

# zsh
echo 'eval "$(vault completions init zsh)"' >> ~/.zshrc

# fish
vault completions init fish > ~/.config/fish/completions/vault.fish
```

## Verify the install

Two commands confirm the binary is on `PATH`, executable, and that the
shipped completions / man page work:

```bash
vault --version
vault --help | head
vault -C fixtures/basic validate --summary --format json
```

You can also confirm the shipped artifacts directly:

```bash
# completions install path (varies by shell):
ls "$(dirname "$(command -v vault)")"/../share/zsh/site-functions/_vault \
   "$(dirname "$(command -v vault)")"/../share/bash-completion/completions/vault \
   2>/dev/null

# man page:
man vault | head
```

If `man vault` reports "No manual entry", the installer placed the man
page at `$CARGO_HOME/share/man/man1/vault.1`. Add that directory to
`$MANPATH`, or invoke directly:

```bash
man "$CARGO_HOME/share/man/man1/vault.1"
```

## Troubleshooting

### Why `curl | sh`?

The shell installer is the official cargo-dist-generated installer for
the release tag. It does the same thing a static binary download would
do, plus completions/man-page placement. If you are uncomfortable
piping into `sh`, use the "download first, inspect, then run" form
above, or use the Cargo build-from-source path below.

### Binary not on PATH

The installer places the binary at `$CARGO_HOME/bin/vault`
(default: `~/.cargo/bin`). If your shell does not already include that
directory in `PATH`, add this to your shell rc file:

```bash
export PATH="$HOME/.cargo/bin:$PATH"
```

### Completions not loaded

Run `vault completions install` (see the [Shell completions](#shell-completions) section above) to wire completions into your shell config in one command. Restart your shell afterward. zsh users may also need `compinit` in their config.

### macOS quarantine warning

Binaries downloaded outside the Mac App Store may be quarantined. Run `xattr -d com.apple.quarantine /path/to/vault` once.

### Cargo build fails with MSRV error

Update your Rust toolchain to the latest stable: `rustup update stable`.

For developer-facing builds (cloning, contributing), see [development.md](development.md).
