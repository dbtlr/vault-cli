# Installation

`norn` ships pre-built binaries for macOS (x86_64 + aarch64) and Linux (x86_64 + aarch64, musl). Windows is not currently supported.

## Shell installer (recommended)

The fastest path is to pipe the release installer into `sh`:

```bash
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/dbtlr/norn/releases/latest/download/norn-run-installer.sh \
  | sh
```

The installer downloads the right binary for your platform, places it in
`~/.cargo/bin` (or `$CARGO_HOME/bin` if set), and installs the shell
completions and man page next to it.

### Safer: download first, inspect, then run

If you would rather review the installer before executing it (always a
good idea for `curl | sh`-style installers), download it and run it as a
second step:

```bash
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/dbtlr/norn/releases/latest/download/norn-run-installer.sh \
  -o norn-run-installer.sh
less norn-run-installer.sh   # review
sh norn-run-installer.sh
```

The installer is generated by [cargo-dist](https://opensource.axo.dev/cargo-dist/);
the source code that produces it is open and reproducible from the tag.

## Cargo

If you have a Rust toolchain installed, build from source:

```bash
cargo install --git https://github.com/dbtlr/norn norn-run
```

Or pin a specific tag:

```bash
cargo install --git https://github.com/dbtlr/norn --tag v0.26.1 norn-run
```

`norn` tracks the latest stable Rust toolchain (see `mise.toml`). See `docs/development.md` for the full MSRV policy.

## Manual binary download

Releases include pre-built archives at:

`https://github.com/dbtlr/norn/releases/`

Each release ships `norn-run-<target>.tar.xz` containing the `norn` binary, shell completions (`norn.bash`, `_norn`, `norn.fish`), and the man page (`norn.1`). Extract, place `norn` on your `PATH`, and copy the completions and man page into your system's appropriate directories.

## Shell completions

After install, enable completions for your shell with:

```bash
norn completions install
```

`norn completions install` auto-detects your shell from `$SHELL` and writes the right configuration. Supported shells: **bash**, **zsh**, **fish**, **powershell**, **elvish**, **nushell**.

To preview what it would write without modifying anything:

```bash
norn completions install --print
```

To replace an existing install (after upgrading norn, or to reset a manual edit):

```bash
norn completions install --force
```

To install for a specific shell rather than the one detected:

```bash
norn completions install zsh
```

### After a norn upgrade

The bash, zsh, powershell, and elvish installers write an `eval` line that re-evaluates the current `norn completions init <shell>` output every shell startup. New completions pick up automatically.

The fish and nushell installers write a script file (because those shells load completions from files, not eval). After upgrading norn, re-run `norn completions install fish` or `norn completions install nushell` to refresh the script with the new norn's completions.

### macOS bash target file

On macOS, the bash installer prefers `~/.bash_profile` over `~/.bashrc` when the former exists, since macOS launches Terminal sessions as login shells. If you use a non-standard bash setup, override the target with `BASH_ENV=/path/to/file norn completions install bash`.

### Manual install

If you prefer to wire completions in by hand, `norn completions init <shell>` emits the script to stdout. Pipe it where your shell expects it:

```bash
# bash
echo 'eval "$(norn completions init bash)"' >> ~/.bashrc

# zsh
echo 'eval "$(norn completions init zsh)"' >> ~/.zshrc

# fish
norn completions init fish > ~/.config/fish/completions/norn.fish
```

## Verify the install

Two commands confirm the binary is on `PATH`, executable, and that the
shipped completions / man page work:

```bash
norn --version
norn --help | head
norn -C fixtures/basic validate --summary --format json
```

You can also confirm the shipped artifacts directly:

```bash
# completions install path (varies by shell):
ls "$(dirname "$(command -v norn)")"/../share/zsh/site-functions/_norn \
   "$(dirname "$(command -v norn)")"/../share/bash-completion/completions/norn \
   2>/dev/null

# man page:
man norn | head
```

If `man norn` reports "No manual entry", the installer placed the man
page at `$CARGO_HOME/share/man/man1/norn.1`. Add that directory to
`$MANPATH`, or invoke directly:

```bash
man "$CARGO_HOME/share/man/man1/norn.1"
```

## Troubleshooting

### Why `curl | sh`?

The shell installer is the official cargo-dist-generated installer for
the release tag. It does the same thing a static binary download would
do, plus completions/man-page placement. If you are uncomfortable
piping into `sh`, use the "download first, inspect, then run" form
above, or use the Cargo build-from-source path below.

### Binary not on PATH

The installer places the binary at `$CARGO_HOME/bin/norn`
(default: `~/.cargo/bin`). If your shell does not already include that
directory in `PATH`, add this to your shell rc file:

```bash
export PATH="$HOME/.cargo/bin:$PATH"
```

### Completions not loaded

Run `norn completions install` (see the [Shell completions](#shell-completions) section above) to wire completions into your shell config in one command. Restart your shell afterward. zsh users may also need `compinit` in their config.

### macOS quarantine warning

Binaries downloaded outside the Mac App Store may be quarantined. Run `xattr -d com.apple.quarantine /path/to/norn` once.

### Cargo build fails with MSRV error

Update your Rust toolchain to the latest stable: `rustup update stable`.

For developer-facing builds (cloning, contributing), see [development.md](development.md).
